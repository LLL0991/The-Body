# The Body 部署上线指南

部署后你会得到一个**固定网址**（如 `https://xxx.railway.app`），手机和电脑打开即可使用，无需再开终端。

---

## 方式一：Railway 部署（推荐）

[Railway](https://railway.app) 支持 Node 项目，有免费额度，适合个人使用。

### 1. 注册与安装

1. 打开 https://railway.app ，用 GitHub 登录。
2. 可选：安装 [Railway CLI](https://docs.railway.app/develop/cli)（用网页也可以完成部署）。

### 2. 把代码放到 GitHub

下面所有带 `git` 的命令都是在**你电脑上的终端**里执行（Cursor 里的 Terminal、VS Code 终端、或 Mac 的「终端」应用都可以），不是在 GitHub 网页上操作。

**步骤 A：在电脑终端里做**

1. 打开终端，进入项目目录并初始化 Git、做第一次提交：

```bash
cd "/Users/lrdliang/Desktop/The Body"
git init
git add .
git commit -m "Initial commit"
```

2. 确保 `.env` 不会被提交（避免密钥泄露）。若项目里已有 `.gitignore` 且包含 `.env` 可跳过；否则在终端执行：

```bash
echo ".env" >> .gitignore
git add .gitignore
git commit -m "Ignore .env"
```

**步骤 B：在 GitHub 网页上做**

3. 打开 https://github.com/new ，新建一个仓库：
   - **Repository name** 随便起（如 `the-body`）；
   - 不要勾选 "Add a README"（保持空仓库）；
   - 点 **Create repository**。

4. 创建完成后，GitHub 会显示一个仓库地址，形如：`https://github.com/你的用户名/the-body.git`，先复制这个地址。

**步骤 C：回到电脑终端**

5. 在**同一个项目目录**下，把本地仓库和 GitHub 连起来并推送（把下面的地址换成你刚复制的）：

```bash
git remote add origin https://github.com/你的用户名/the-body.git
git branch -M main
git push -u origin main
```

输入 GitHub 用户名和密码（或 Personal Access Token）后，代码就上传到 GitHub 了。

### 3. 在 Railway 创建项目

1. 登录 https://railway.app → **New Project**。
2. 选 **Deploy from GitHub repo**，授权并选择你的仓库（如 `the-body`）。
3. 选中仓库后，Railway 会拉取代码并尝试构建。

### 4. 配置构建与启动命令

在 Railway 项目里点进该服务 → **Settings**：

- **Build Command** 填：`npm install && npm run build`
- **Start Command** 填：`npm run start`
- **Root Directory** 留空（或填 `/`）。

保存后触发一次 **Redeploy**。

### 5. 配置环境变量

同一服务 → **Variables**，添加以下变量（值按你的实际情况填）：

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `NODE_ENV` | 固定为 production | `production` |
| `VITE_LLM_API_KEY` | LLM API Key（如硅基流动） | `sk-xxx` |
| `VITE_LLM_BASE_URL` | LLM 接口地址 | `https://api.siliconflow.cn/v1` |
| `VITE_LLM_MODEL` | 模型名 | `Qwen/Qwen2.5-7B-Instruct` |
| `VITE_LLM_USE_PROXY` | 必须为 true（走同源代理） | `true` |
| `SILICONFLOW_API_KEY` | 语音/拍照用（可与上面同 Key） | `sk-xxx` |
| `VITE_LLM_VISION_MODEL` | 可选，拍照识图模型 | `Qwen/Qwen2.5-VL-7B-Instruct` |

**重要**：`VITE_LLM_USE_PROXY` 必须设为 `true`，前端才会通过你的域名访问 LLM，避免跨域和 Key 暴露。

保存后 Railway 会重新构建并部署（构建时会把这些 `VITE_*` 打进前端）。

### 6. 生成公网地址

在服务 **Settings** 里找到 **Networking** → **Generate Domain**，会得到一个类似 `https://the-body-production-xxxx.up.railway.app` 的地址。

用浏览器或手机打开该地址即可使用。

### 7. 后续更新

改完代码后：

```bash
git add .
git commit -m "Update"
git push
```

Railway 会自动重新构建并部署。

## 方式二：Render 部署

1. 打开 https://render.com ，用 GitHub 登录。
2. **New** → **Web Service**，选你的仓库。
3. 配置：
   - **Build Command**：`npm install && npm run build`
   - **Start Command**：`npm run start`
   - **Instance Type**：选免费档即可。
4. 在 **Environment** 里添加与上面相同的环境变量（含 `VITE_LLM_USE_PROXY=true`）。
5. 创建后等部署完成，Render 会给你一个 `https://xxx.onrender.com` 的地址。

---

## 部署后检查

- 打开你的网址，能正常看到首页。
- 在输入框输入「中午吃了一碗拉面」→ 点记录 → 能出现解析结果并确认记录。
- 若已配置 `SILICONFLOW_API_KEY`，语音、拍照也应可用。

若文字解析报错，请检查：

1. 环境变量里 `VITE_LLM_USE_PROXY` 是否为 `true`。
2. `VITE_LLM_API_KEY`、`VITE_LLM_BASE_URL`、`VITE_LLM_MODEL` 是否在 Railway/Render 的 Variables 里正确填写并已重新部署。

## 本地先验证再部署

在本地模拟生产环境，确认无误后再推到 Railway/Render：

```bash
cd "/Users/lrdliang/Desktop/The Body"
npm run build
NODE_ENV=production npm run start
```

浏览器打开 http://localhost:5175（或终端里提示的端口），能正常使用文字/语音/拍照后，再按上面步骤部署。
