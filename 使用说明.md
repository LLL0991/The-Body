# The Body 使用说明

## 一、环境准备

### 1. 安装依赖（首次或拉新代码后）

```bash
cd "/Users/lrdliang/Desktop/The Body"
npm install
```

### 2. 配置 API（必做）

复制环境变量示例并编辑：

```bash
cp .env.example .env
# 用编辑器打开 .env，取消注释你要用的一组并填入 Key
```

**最少需要配置：**

| 用途         | 变量名                 | 说明 |
|--------------|------------------------|------|
| 文字/营养解析 | `VITE_LLM_API_KEY`      | LLM 的 API Key（你已用硅基流动） |
| 文字/营养解析 | `VITE_LLM_BASE_URL`     | 如 `https://api.siliconflow.cn/v1` |
| 文字/营养解析 | `VITE_LLM_MODEL`        | 如 `Qwen/Qwen2.5-7B-Instruct` |
| 语音识别 + 拍照识别 | `SILICONFLOW_API_KEY` 或沿用 `VITE_LLM_API_KEY` | 语音转文字、照片描述都用硅基流动 |

国内网络建议在 `.env` 里加上：

```env
VITE_LLM_USE_PROXY=true
```

这样浏览器会走本地 Vite 代理请求 LLM，避免跨域。

## 二、正式使用（本地）— 一个命令、一个网址

只开**一个终端**，运行：

```bash
cd "/Users/lrdliang/Desktop/The Body"
npm run dev
```

会同时启动：前端页面 + 语音/拍照服务。然后：

- **电脑**：浏览器打开 **http://localhost:5173**
- **手机**：和电脑连同一 WiFi，在终端里看 `Network: http://192.168.x.x:5173`，用这个地址在手机浏览器打开

所有功能（文字解析、语音、拍照）都通过这一个网址即可使用，无需再开第二个终端。

（若只想单独跑前端或单独跑语音服务，可用 `npm run dev:only` 或 `npm run voice-server`。）

## 三、怎么用

### 首页（一级页面）

1. **文字记录**  
   在输入框输入例如「中午吃了一碗兰州拉面加白切肉」→ 点「记录」→ 看解析结果（食材、克数、P/C/F）→ 选要记到哪一餐 → 点「确认」完成记录。

2. **语音记录**  
   点麦克风 → 再点一次结束录音 → 等识别出文字后，流程同文字：看解析 → 选餐次 → 点「确认」。

3. **拍照记录**  
   点拍照/上传图标 → 选照片 → 等 AI 描述 → 再走解析与确认（同上）。

4. **生重/熟重**  
   需要按生重记（如生面、生米）时，勾选「记录为生重」再点确认。

### 详情页（点底部某一餐进入）

- 查看/编辑该餐已有食材和克数。
- 同样支持：文字输入、语音、拍照；可切换显示为「生重」或「熟重」。

### 碳水提示

- 当日碳水超过目标会红字提示「今日碳水超标」。
- 所剩不多时会黄字提示「碳水所剩不多」，建议后续低碳。

## 四、想要「固定网址、不跑终端」怎么办？（部署上线）

如果希望像普通网站一样，**不运行任何命令，随时点开一个网址**就用，需要把应用部署到云端，拿到一个固定 URL（例如 `https://xxx.railway.app`）。

可选方式示例：

1. **Railway / Render / Fly.io**  
   把本项目部署为一个 Node 应用：用 `server/voiceServer.mjs` 提供接口，并托管前端打包后的静态文件（或再配一个静态站）。在平台里配置环境变量（`VITE_LLM_API_KEY`、`SILICONFLOW_API_KEY` 等），部署完成后会得到一个固定网址，手机、电脑直接打开即可。

2. **仅前端部署（Vercel / Netlify 等）**  
   若只部署前端，文字解析需通过 serverless 或自建接口代理 LLM；语音和拍照需单独部署 `voiceServer.mjs` 到支持 Node 的平台，并在前端设置 `VITE_VOICE_AND_IMAGE_API` 为该服务地址。

本地生产构建：

```bash
npm run build
```

生成在 `dist/`。部署时需同时提供 LLM 代理与语音/拍照接口，参见上文。

## 五、常见问题

| 现象 | 处理 |
|------|------|
| 文字解析没反应 / 报错 | 检查 `.env` 里 `VITE_LLM_API_KEY`、`VITE_LLM_BASE_URL`、`VITE_LLM_MODEL` 是否正确；国内可开 `VITE_LLM_USE_PROXY=true`。 |
| 语音识别失败 | 确认已运行 `npm run voice-server`，且 `.env` 里配置了 `SILICONFLOW_API_KEY` 或 `VITE_LLM_API_KEY`。 |
| 拍照识别失败 | 同上，语音服务要开着；硅基流动需用支持视觉的模型（如 Qwen2.5-VL），可在 `.env` 设 `VITE_LLM_VISION_MODEL`。 |
| 手机打不开 | 手机和电脑同一 WiFi；用电脑 IP:5173，不要用 localhost；手机勿开 VPN 连国外。 |

---

总结：**日常使用 = 一个终端运行 `npm run dev`，浏览器打开 http://localhost:5173（或手机用同 WiFi 下的电脑 IP:5173），用文字/语音/拍照记录即可。**
